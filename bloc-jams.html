BLOC JAMS

DEEP(ish) DIVE

Bloc Jams is a digital music player, much like Spotify or Amazon Music.

WHAT IS IT?

This project was a part of the curriculum at Bloc in front-end web development. Bloc Jams was built twice to experiment with technology usability and to facilitate the understanding of front-end web technologies. The first iteration of this project was built solely using HTML and CSS for the layout and responsiveness, with Javascript and jQuery implementing user-interactivity.

The second version was built using AngularJS in contrast. The views that are loaded are still built with HTML and CSS for layout and responsiveness, but user interactive and dynamic information are handled using Javascript and Angular directives.

UNDER THE HOOD AND NOTABLE

When the collection page is loaded, the view is populated with albums available to listen to. For the sake of this project, the album information is stored inside a static fixtures file. The information contained within this file is JSON data. As seen in the demo, the first version of this project requests the first item in the JSON object and repeats the information by appending an album "template" using a for loop. The AngularJS version of Bloc Jams works similarly, however, the template is repeated using the ng-repeat directive.

It is worth noting, that this system of displaying the albums in the collection view could be modified to query a server side database of albums as well. Using the same for-loop or ng-repeat directive implemented in this example, the new album data can be used instead seamlessly.


The individual album view utilizes a similar method in the track listing. The individual song names and associated information reside in the album JSON object as well. When the page is loaded, the template for the individual songs are loaded using a for-loop that iterates based on the number of songs on the album. In similar fashion to the collection page, the AngularJS version of Bloc Jams uses the ng-repeat directive to display the individual tracks.

THE ANGULAR WAY

Throughout the development of Bloc Jams, the benefits of AngularJS became most apparent when working with the track listing on the individual album page. This presented itself as the most complex behavior implemented in Bloc Jams.

The behavior of the individual tracks needed to be as follows:

  1. When the user clicks a song to play it, the song number should change to a pause button.
  2. When the mouse leaves the table row of the currently playing song, the pause button should remain.
  3. When the user switches songs, the previously playing song's table cell should revert the content back to the song number.
  4. When the user hovers over each of the songs that aren't playing, the play button should still appear; however, the application shouldn't show the play button when the user hovers over the playing song.

THE TAKE AWAY

I began this project with assumptions that proved ill-got or misguided.

Having used jQuery to varying small degrees, I thought using it would result in the easiest implementations of the features I needed. I grossly underestimated the power of AngularJS. As I began to utilize more of the framework's built-in functionality, I was shocked at the ease of functionality implementation. AngularJS proved to be every bit as powerful as I have understood it to be, and it made refactoring Bloc Jams an exciting undertaking in learning a new web technology.
