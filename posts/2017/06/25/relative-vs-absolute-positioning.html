This particular writing assignment dealt with the differences between relative and absolute positioning in CSS. The reason why I am featuring this particular article, is becuase I found some of the nitty-gritty details about these two attributes much more interesting than I expected them to be.

Relative and absolute positioning share many characteristics but differ in a major way.

The default value of the position attribute is static. An element positioned with position: static; will fall into the flow the document and behave in a manner similar to how stacking physical blocks in the real world would. Statically positioned elements, however, cannot be moved with offset properties such as top right left and bottom. This is why one may want to position an element relatively using position: relative;. Elements positioned with position: relative; will still fall into the flow of the document, like stacking blocks on top of each other, but can be positioned with offset attributes.

This is in distinct contrast to positioning an element absolutely with position: absolute;. An element positioned with position: absolute; will be removed from the flow of the document. This means that it does not affect nor is affected by other elements on the page. In fact, if you were to have three elements positioned statically or relatively to each other and set the middle element to have a position of absolute, the third and bottom item would slide up and cover the second item. In this instance, the third item is no longer aware of the second item because it has been removed from the flow of the document. See the following examples.
